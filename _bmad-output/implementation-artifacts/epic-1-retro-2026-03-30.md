## 3. Challenges and friction (systems, not blame)

1. **Intentional deferrals stacked up** — Many review findings were correctly deferred to Epic 5 (resilience) or Epic 6 (deployment). That was the right trade-off, but it requires **discipline** to track them in `deferred-work.md` so nothing silently rots.

2. **Spec vs implementation tension** — Example: health endpoint NFR14 (500ms) vs a longer HTTP timeout for the model probe. Documented as inconsistent; resolution was deferred rather than ignored.

3. **Operational assumptions** — Single-worker uvicorn, CWD-relative DB/cache paths, and multi-worker CSV load races were called out; acceptable for PoC, not invisible for a future scale-out.

4. **Test isolation** — Notes about tests touching a live `freightmind.db` path predate several stories; worth tightening when touching those tests, not necessarily blocking Epic 1 closure.

5. **`main.py` model imports** — Tables only get created for models imported into startup. New SQLAlchemy models must be wired in manually; a small registry pattern was suggested for later.
