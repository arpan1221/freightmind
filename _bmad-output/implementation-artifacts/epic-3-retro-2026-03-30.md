## 3. Deep story analysis — themes

### What went well

- **Consistent agent pattern:** Story 3.1 dev notes document a clear pipeline (`ExtractionPlanner` → `ExtractionExecutor` → `ExtractionVerifier` → DB), aligned with Epic 2 and easy to test in layers.
- **Incremental complexity:** 3.1 laid plumbing; 3.2 normalisation, 3.3 confidence, and 3.4–3.6 lifecycle endpoints built without rewriting the core.
- **Standalone invocability (3.8):** Parity with Epic 2’s 2.7 story — extraction can be developed and tested without pulling analytics imports.
- **Automated tests:** Dedicated `test_story_3_*` files cover planner, executor, verifier, and routes with mocks, matching project conventions.

### Struggles and friction (systems, not individuals)

- **HTTP semantics vs. POC responses:** Deferred work records that `POST /extract` returns HTTP 200 for unsupported file types and some failures, with errors in the body — workable for a POC but confusing for API clients and observability.
- **Error surface area:** Raw `message=str(e)` on extraction failures and broad `except` paths were flagged in reviews; Epic 5 later centralised `ErrorResponse` (expected sequencing).
- **Duplication and drift risk:** `_HEADER_FIELDS` vs `_ALLOWED_CORRECTION_FIELDS`, router namespace split (`/api/documents` vs `/api/extract`-style paths for cancel) — manageable but noted in multiple story reviews.
- **Normalisation edge cases:** Locale (decimal comma), ISO datetime-with-time, two-digit years, OCR whitespace — documented as out-of-scope tradeoffs with follow-up if product needs stricter data quality.
- **Frontend polish:** Upload panel review called out MIME trust (browser `file.type`), line-item `key={index}`, and a11y gaps — acceptable for POC, listed for hardening.

### Technical debt carried forward (from stories + `deferred-work.md`)

| Area | Examples |
|------|-----------|
| Upload safety | No `MAX_UPLOAD_BYTES`; full file read into memory |
| PDF scope | Single-page (page 0) only; multi-page deferred |
| Persistence | `confirmed_by_user` magic 0/1; optional boolean/enum later |
| List API | `GET /extractions` unpaginated |
| Concurrency | TOCTOU on confirm; no `SELECT FOR UPDATE` in POC |
| Pooling | `ModelClient` per request (same theme as Epic 2) |

None of these blocked Epic 4’s linkage stories in the plan; several are explicitly revisited under Epic 5–6 or production hardening.
