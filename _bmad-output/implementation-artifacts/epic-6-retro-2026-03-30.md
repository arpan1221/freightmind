## 3. Deep story analysis — patterns

### What went well

- **Docs and code alignment:** Story 6.1 explicitly reconciled architecture snippets with repo-authoritative `env_file: .env` and `NEXT_PUBLIC_BACKEND_URL` for host-browser calls — reduced “works in compose but README lies” risk before 6.5.
- **Guardrails where automation is thin:** Compose contract tests (`test_story_6_1.py`), Dockerfile `PORT` assertions (`test_story_6_2.py`), `getApiBaseUrl` Vitest coverage, and demo-invoice inventory/README tests caught regressions without requiring live Render/Vercel in CI.
- **Naming friction resolved in product:** Epic text said `NEXT_PUBLIC_API_URL`; the codebase already used `NEXT_PUBLIC_BACKEND_URL`. Story 6.3 implemented precedence + fallbacks instead of breaking Docker — good compatibility story for monorepos.
- **Review loops caught real issues:** `exec` before `uvicorn` for signal handling (6.2); empty/whitespace env treated as unset for API base URL (6.3); README manifest assertions for linkage wording (6.4); README cross-links and journey-5 non-determinism note (6.5).
- **Evaluator package:** Demo assets + generator script + five-journey README give a repeatable path for humans who will not read `app/` first.

### Struggles and friction (systems, not blame)

| Theme | Evidence |
|--------|-----------|
| **Operator-validated vs CI-proven** | Stories 6.2–6.3 and NFR6/NFR12 live checks are intentionally runbook/deployment-dependent; the repo encodes *how* to verify, not a recorded production URL or cold-start trace. |
| **Build-time public env** | `NEXT_PUBLIC_*` requires image rebuild or Vercel redeploy when the Render URL changes — documented repeatedly; still a common footgun for new contributors. |
| **Health semantics** | `status: ok` needs DB + OpenRouter reachability; degraded paths are valid in tests but demos want `ok` — called out in 6.2 dev notes. |
| **Vision non-determinism** | Demo invoices target LOW/NOT_FOUND for specific fields; actual model output may vary — README and 6.4 defer empirical re-check to operator with a live key. |
| **Stale cross-references** | 6.2’s “Previous story intelligence” once referenced 6.1 as not implemented — story files are snapshots; sprint-status shows correct order. |

### Technical debt and deferred follow-ups (from stories + reviews)

| Area | Notes |
|------|--------|
| **Container hardening** | Non-root user in Dockerfile — still deferred from Epic 1 / 6.2 reviews. |
| **Render data** | Default SQLite is ephemeral unless persistent disk + `DATABASE_URL` — acceptable for PoC; called out in 6.2. |
| **Generator script** | `generate_demo_invoices.py` error handling could be friendlier on disk/permission failures — low priority. |
| **CI** | No automated post-deploy smoke; optional future: link checker for README or staging URL checks if the team adds infrastructure. |
