## 3. What went well

**Bob (Scrum Master):** The team treated **prompt layers as a system**: changing only `analytics_sql_gen` without aligning planner intent would have failed FR25; story files explicitly called out all three prompt files plus the answer prompt where needed.

**Charlie (Senior Dev):** **4.2’s comment-stripping fix** for cross-table detection was a good catch — SQL comments could falsely imply both tables were referenced. That is exactly the kind of edge case that breaks trust in “linkage” UX.

**Dana (QA Engineer):** Tests stayed **contract-focused**: mocked LLM returning fixed SQL, assertions on response shape and executed SQL, plus UI tests for full SQL string and `<details>` default collapsed.

**Alice (Product Owner):** **4.3** validated an important product principle: we did not add a separate “display SQL” field; transparency stays **what actually ran**.

**Elena (Junior Dev):** **4.1** review iterations on `_DOC_QUESTION_PATTERNS` and copy aligned with the **Documents** tab in the app — small but reduces analyst confusion.

**Winston (Architect):** Dependencies on Epics 2 and 3 held: raw SQL execution path unchanged, `confirmed_by_user` semantics consistent with confirm/cancel flow from Epic 3.

## 4. Challenges and friction (systems, not blame)

1. **Pipeline ordering (4.1)** — Intent classification can run before the no-confirmed-extractions short-circuit. Review noted that if the planner marks a document question `out_of_scope`, the user might not see the friendly empty-state message. **Mitigation:** dual-table planner prompts; full reorder deferred as a product decision.

2. **Null exclusions vs linkage (4.2)** — `_count_null_exclusions` remains **shipments-oriented** by design for cross-table runs; documented so the answer layer does not silently claim exclusions that were not computed for `extracted_documents`.

3. **NFR1 (15s)** — Not asserted in automated tests; treated as operational/PoC alignment with timeouts and bounded work.

4. **Epic 3 retrospective** — No `epic-3-retro-*.md` exists, so lessons from extraction epic were carried only via story files and code, not a formal retro paper trail.
