Rules:
1. Output ONLY the raw SQL — no markdown, no code fences, no explanation
2. Use only read-only SELECT (including UNION ALL / subqueries) — never DROP, DELETE, UPDATE, INSERT, ALTER, CREATE, REPLACE, ATTACH, PRAGMA, EXEC
3. For **shipments**: add WHERE freight_cost_usd IS NOT NULL when computing averages or sums on freight cost; same for weight_kg when aggregating weight
4. Use strftime('%Y', scheduled_delivery_date) for year-based grouping on **shipments** date columns
5. Limit result rows to 100 unless the user asks for all records
6. Use exact column names as listed — do not invent column names
7. For linkage across sources, align **destination_country** / **shipment_mode** vocabulary with **shipments** when comparing to the historical dataset

8. **Cross-table comparison (Story 4.2 / FR27):** When the user compares or combines **historical SCMS shipments** with **their confirmed uploads**, emit a single statement that references **both** `shipments` and `extracted_documents` using **UNION ALL** (often with a literal `source` or `dataset` column) or a **JOIN** on comparable dimensions (e.g. `shipment_mode`, `destination_country`). Filter confirmed uploads with `confirmed_by_user = 1` unless they ask for pending rows.

Example pattern (adapt column names and filters to the question):
SELECT 'dataset' AS source, AVG(freight_cost_usd) AS avg_freight
FROM shipments WHERE shipment_mode = 'Air' AND freight_cost_usd IS NOT NULL
UNION ALL
SELECT 'extracted' AS source, AVG(total_freight_cost_usd)
FROM extracted_documents WHERE shipment_mode = 'Air' AND confirmed_by_user = 1

9. Prefer **UNION ALL** when returning parallel aggregates from two tables; use **JOIN** when the question needs a single merged row set with aligned keys.
